# Accesibilidad — AutonomIA

Estado y criterios de regresión para la web pública (`site/`). Objetivo: **WCAG 2.2 nivel AA** sin issues "critical/serious" en axe-core y Lighthouse a11y ≥ 95.

## Estado actual (2026-05-10, ITEAA-1575)

Cambios incluidos en esta auditoría:

- **Skip-link "Saltar al contenido"** en todas las páginas (`site/index.html`, `site/cuestionario.html`, `site/servicios.html`, `site/blog/`, `site/aprende/`, `site/blog/_template.html`). Aparece visible al recibir foco con teclado y salta a `<main id="main-content" tabindex="-1">`. **WCAG 2.4.1 Bypass Blocks**.
- **Contraste de texto**:
  - `--accent`: `#6366f1` → `#818cf8` (indigo-400). Texto `var(--accent)` sobre `var(--bg)` y `var(--card)` pasa de **3.6:1 (FAIL)** a **5.35:1 (AA)**.
  - `--accent-fg`: `#ffffff` → `#0b0d12`. Texto blanco sobre indigo-400 daba 3.26:1 (fail). Texto oscuro sobre indigo-400 da 5.35:1 (pasa AA). Afecta `.cta--primary`, `.intro__avatar`, `.skip-link`.
  - `--muted`: `#9aa1b2` → `#b4bccd`. Sube de ~6.5:1 a ~8.6:1 sobre fondos oscuros (margen extra para texto pequeño 0.8–0.85rem). **WCAG 1.4.3 Contrast (Minimum)**.
  - `.cta--ghost` border: `rgba(255,255,255,0.20)` → `rgba(255,255,255,0.45)` para superar el ratio 3:1 de elementos UI no textuales. **WCAG 1.4.11**.
- **Foco visible** (`:focus-visible`) en todos los elementos interactivos (`a`, `button`, `summary`, `.cta`): anillo de 2px `var(--fg)` con offset 2px. **WCAG 2.4.7 Focus Visible** y la nueva **2.4.11 Focus Not Obscured (AA)** de WCAG 2.2.
- **Cuestionario** (`site/cuestionario.html`):
  - Región `role="alert" aria-live="assertive"` (`#quiz-error`) que anuncia errores de validación con el texto de la pregunta concreta que falta.
  - `aria-invalid="true"` en el campo problemático.
  - Borrado del error y del marcado visual al cambiar de paso (`showStep`).
  - `<nav class="quiz__nav" aria-label="Navegación del cuestionario">`.
  - `aria-live="polite"` ya estaba en `#step-indicator` y en el resultado (`#result`, `#result-package`).
- **Modal de intro** (`site/assets/intro.js`): cierre con `Escape`/`Enter`/`Space` confirmado (`intro.js:117`); también respeta `prefers-reduced-motion` y `webdriver` (los bots/Playwright se saltan la animación).
- **Imágenes**: no hay `<img>` en HTML. La única imagen es OpenGraph (`assets/og-default.png`) y ya tiene `og:image:alt`/`twitter:image:alt`.
- **Estructura semántica**: todas las páginas tienen `<html lang="es">`, `<header>`, `<nav>`, `<main>`, `<footer>`, jerarquía de `<h1>` única por página.

## Criterios de regresión

Cualquier PR que toque `site/**` debe respetar:

1. **No bajar el contraste de texto.** Si necesitas un color nuevo, comprueba que el ratio sobre `--bg` (`#0b0d12`) y `--card` (`#141823`) supere 4.5:1 para texto normal y 3:1 para texto grande (≥18.66pt o ≥14pt bold). Para fondos translúcidos (`rgba(...)`), comprueba el peor caso.
2. **No introducir `<main>` sin `id="main-content" tabindex="-1"`** (rompería el skip-link).
3. **No quitar el skip-link** (`<a class="skip-link" href="#main-content">`) del primer hijo de `<body>`.
4. **No usar `outline: none`** sin un sustituto visible (`:focus-visible` con borde, sombra o cambio de color de al menos 3:1 contra el adyacente).
5. **No introducir un nuevo `<fieldset>`/grupo de radios sin `<legend>`** o sin un `<label>` por opción.
6. **Los formularios validados con JS deben anunciar el error** vía `role="alert"` o región `aria-live` (no sólo `outline` rojo + `focus()`).
7. **Las animaciones deben respetar `prefers-reduced-motion: reduce`** (revisar bloques `@media` en `assets/style.css` si introduces transiciones nuevas).
8. **Las imágenes nuevas requieren `alt`** descriptivo (o `alt=""` si son puramente decorativas).
9. **Los iconos hechos con `::before`/símbolos** (✓, ☐, "+") están envueltos en CSS — si los traes al HTML, márcalos `aria-hidden="true"` y deja un texto accesible al lado.

## Cómo verificar localmente

```bash
# Servir el sitio
python3 -m http.server -d site 8080

# axe-core (CLI, requiere Node y Chrome instalado)
npx -y @axe-core/cli http://localhost:8080/ \
  http://localhost:8080/cuestionario.html \
  http://localhost:8080/blog/ \
  http://localhost:8080/blog/que-puede-hacer-la-ia-en-mi-pyme.html \
  --tags wcag2a,wcag2aa,wcag21a,wcag21aa,wcag22aa

# Lighthouse a11y
npx -y lighthouse http://localhost:8080/ \
  --only-categories=accessibility --quiet --chrome-flags="--headless"
```

Verificación manual mínima por página antes de mergear:

- [ ] Pulsa `Tab` desde la URL: el primer foco visible debe ser el "Saltar al contenido".
- [ ] `Shift+Tab` no debe quedarse atrapado en el overlay de intro (`Escape` lo cierra).
- [ ] Cada CTA (botón o `a.cta`) muestra anillo de foco al recibirlo por teclado.
- [ ] En `cuestionario.html`: pulsar "Siguiente" sin responder anuncia (con lector) "Falta responder: …" y mueve el foco a la pregunta.
